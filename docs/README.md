# OpsHub Docs

This folder is the operating harness for PhongVu OpsHub. It keeps product truth,
story-sized work, decisions, and validation proof close to the code.

## Structure

| Path | Purpose |
| --- | --- |
| `product/` | Current product contracts for accepted behavior |
| `stories/` | Story packets, backlog, and implementation evidence |
| `decisions/` | Durable architecture and product tradeoffs |
| `templates/` | Reusable story, decision, intake, and validation templates |
| `TEST_MATRIX.md` | Behavior-to-proof control panel |
| `FEATURE_INTAKE.md` | Risk classification before code changes |
| `ARCHITECTURE.md` | Current architecture map and boundary rules |
| `HARNESS.md` | Human-agent collaboration loop |
| `HARNESS_BACKLOG.md` | Process improvements found during work |

## Rule

Do not let docs become a second app. Keep them short, current, and useful for
the next change.
